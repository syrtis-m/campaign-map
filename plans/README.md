## Point-crawl connections (the requested feature)

Split into 004 (make connections declared in note frontmatter render as themed
lines, compatible with every map style) and 005 (create/remove them from the
map UI). The design is canon-native: a connection is a `connections:` list in a
location note's frontmatter, resolved at reconcile time into line features —
same "canon = notes" model as everything else, so links sync, survive rename,
and vanish when an endpoint is deleted. Straight undirected lines in v1;
curved/waypointed paths, directional arrowheads, and travel-time edge labels are
scoped as follow-ups inside those plans.

## Findings considered and rejected

- *Enlarge the visible dots to fix clickability*: rejected in favor of an
  invisible hit-tolerance (plan 001) — fat dots fight the cartographic-discipline
  bar.
- *A single monster "build the full UX" plan*: rejected — decomposed into 003
  (toolbar) + the Direction items, so each piece is independently verifiable.
- *Height-relief overlay in plan 002 using an existing token*: rejected — token
  luminance flips between light/dark themes; relief needs a per-theme token and
  is deferred to a follow-up (documented in 002).
