# 030-B — Prove-by-breaking evidence

Plan 030-B item 2 requires: a live Obsidian gate may be deleted only after its
headless replacement is PROVEN to catch a seeded failure of the thing the gate
uniquely guards. This document records 20 seeded-break experiments run entirely
headlessly (`npx vitest run <files>` / `npx tsc`). Every break was reverted with
`git checkout -- <file>` and the suite re-run green before moving on.

Method per row: (1) locate source surface; (2) smallest type-valid edit that
recreates the failure class; (3) run only the replacement file(s), capture the
failing assertion + excerpt; (4) revert; (5) re-run green. A break that only
trips tsc (not the runtime band) is recorded as NOT CAUGHT for the semantic net
with tsc noted as a secondary catcher.

Legend: **CAUGHT** = named replacement fires on a type-valid regression.
**NOT CAUGHT** = the named net has a hole (a finding, not a task failure).
Net kind is called out: *band/signature/invariant* (strong) vs *golden-diff*
(weaker — catches any change, not the specific property).

## 1. procgen41 — skip regeneration after a geometry edit

- **Gate guarded:** a vertex edit re-adapts the city to the reshaped district.
- **Seeded edit:** `src/controller/MapController.ts` `commitSketchEdit` — removed
  the `await this.generateRegion(after, { force: true })` call in the non-debounce
  procgen-region geometry branch (kept the `cascadeFromRoot` line). Confirmed live:
  `cascadeFromRoot(after.id)` calls `cascadeDownstream([rootId], new Set([rootId]))`,
  whose closure filter (`!done.has(n.id)`) excludes the root — so the root region is
  genuinely never regenerated by the cascade. This is the only root-regen call on
  the geometry-edit path.
- **Replacement run:** `src/controller/MapController.test.ts` (whole file, 48 tests).
- **VERDICT: NOT CAUGHT (semantic net has a hole).** All 48/48 passed with the break
  in place, including the named "adapts the city to a reshaped district while keeping
  its seed" test. Root cause: the test nudges vertex 0 *outward* (`moveVertex(id,0,[8,-28])`),
  which *enlarges* the ring. The stale (pre-edit) cache features are retained in
  `loadedTiles` and remain inside the enlarged ring, so `report.count > 0` and
  `report.outside === 0` both still hold. The seed-persistence and `sketch-edit`-log
  assertions are orthogonal to regeneration. tsc stays clean (type-valid break) so
  even tsc does not catch it.
- **Recommended repair (before deleting the live gate):** strengthen the vertex-edit
  test to prove regeneration actually ran. Any one of: (a) capture
  `host.controller.generatorRunCount` before `moveVertex` and assert it increments;
  (b) nudge a vertex *inward* (shrink the ring) so a skipped regen leaves stale
  features outside → `report.outside > 0` fires; (c) assert the painted feature-id
  set (`regionFeatureIds`) changes across the edit. Until one of these lands, the
  procgen41 live gate's unique coverage is NOT reproduced headlessly.
- **Revert verified green:** `git checkout -- src/controller/MapController.ts`; named
  test re-run → 1 passed / 47 skipped.
